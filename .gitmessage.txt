# 🧵 Commit Convention :: Gitmoji
# 제목 (한 줄 요약)
# 형식: <이모지> <타입>: <변경 사항 요약>
# 예: ✨ feat: 새로운 기능 추가

# 본문 (변경 사항에 대한 상세 설명)
# - 변경된 내용에 대한 설명을 여기에 작성합니다.
# - 여러 줄로 작성할 수 있으며, 각 줄은 72자 이내로 유지하는 것이 좋습니다.
# - 필요시 관련 이슈 번호를 언급할 수 있습니다. 예: Fixes #123

# 푸터 (선택 사항)
# - 추가적인 메모나 참고 사항을 여기에 작성합니다.
# - 예: BREAKING CHANGE: 이전 버전과의 호환성 문제에 대한 설명

# Commit Title Type
# ✨ feat     : 새로운 기능 추가
# 🐛 fix      : 버그 수정
# 💡 chore    : 오타 수정 및 새로운 기능이 추가되지 않고, 코드가 변경 된 경우 (주석 추가 포함)
# 📝 docs     : 문서 수정 (예: README 수정)
# 🚚 build    : 빌드 관련 파일 수정 (패키지 추가/삭제)
# ✅ test     : 테스트 코드 추가 및 수정
# ♻️ refactor : 코드 리팩터링 (기능 변경 없이 구조 개선)
# 🚑 hotfix   : 긴급 수정
